# Бот для учета партий в настольный теннис

Telegram-бот ведет статистику партий: общий счет побед и поражений, мячи, соперников и результаты отдельных партий.

## Что уже есть

- inline-кнопки под сообщениями;
- замена сообщений вместо длинной цепочки новых сообщений;
- удаление лишних сообщений пользователя, если Telegram разрешает удаление;
- меню соперников;
- тестовый соперник для первого запуска;
- добавление результата в формате `11-7`, `11 7`, `15:13`;
- автоматическая разбивка счета больше 11 на `10-10` и овертайм;
- редактирование общего счета партий и общего количества мячей без редактирования конкретной партии;
- дата и время партий сохраняются по Москве;
- хранение в SQLite.

## Почему SQLite, а не Google Sheets

Для личного бота SQLite проще и безопаснее: не нужны сервисные аккаунты Google, права на таблицы и отдельная интеграция с API. На Railway базу нужно хранить на Volume, например по пути `/data/table_tennis.sqlite3`.

## Важный момент про Railway

Railway подходит для такого маленького бота, но это не гарантированно бесплатный навсегда вариант. На официальной странице тарифов сейчас указан free/trial с кредитами и дальнейшими условиями, поэтому перед запуском лучше проверить актуальную страницу: https://railway.com/pricing

## Запуск локально

1. Создайте бота через BotFather и получите токен.
2. Установите зависимости:

```bash
pip install -r requirements.txt
```

3. Запустите:

```bash
BOT_TOKEN=ваш_токен python -m app.bot
```

## Переменные окружения

- `BOT_TOKEN` — токен Telegram-бота.
- `DATABASE_PATH` — путь к SQLite-базе. Для Railway Volume удобно `/data/table_tennis.sqlite3`.
- `SEED_TEST_OPPONENT` — добавлять тестового соперника при первом запуске. По умолчанию `true`.

## Деплой на Railway

1. Создайте проект Railway из репозитория.
2. Добавьте переменную `BOT_TOKEN`.
3. Подключите Volume и смонтируйте его в `/data`.
4. Убедитесь, что `DATABASE_PATH=/data/table_tennis.sqlite3`.
5. Оставьте одну реплику сервиса: бот работает через polling, две реплики могут обработать одно обновление дважды.
6. Railway возьмет команду запуска из `railway.json`.
